Device Distribution

What is Google Android?

A software stack for mobile devices that includes:

  • An operating system
  • Middleware
  • Key Applications

Uses Linux to provide core system services

  • Security
  • Memory management
  • Process management
  • Power management
  • Hardware drivers

Android Apps

  • Built using Java/Kotlin and Android SDK libraries
    • No support for some Java libraries like Swing or AWT
  • Java/Kotlin code compiled into ART byte code (.dex)
    • ART is optimized for mobile devices (better memory management, battery utilization, …)
  • Dalvik VM runs .dex files (not JRE) NOT .class!

Building and Running

  • Compile .kt or .java files into .dex
  • Combine AndroidManifest.xml, assets (images, files, …), and .dex files into a single .apk (App Package)

Applications are Sandboxed

  • By default, each app (and all its components!) run in its own Linux process.

  • By default, each app is assigned a unique Linux user ID.

  • Permissions are set so only your app can view its own files

Jetpack Compose Basics

Intro to Jetpack Compose

Jetpack Compose is Android’s new UI Framework

Composables

UIs are built by combining (or composing) Composable functions, each of which produce an part of the UI

@Composable
fun Hello() {
    Text("Hello World")
}

Declarative vs Imperative

The Compose runtime is responsible for rendering your UI. Your Composable functions define what the UI should look like. Compose is a declarative framework, as opposed to an imperative framework.

Unidirectional data flow

  • Data flows down the view heirarchy.
  • Events flow up the heirarchy.

Idempotency

  • Multiple, identical function calls result have the same effect as a single call.
  • Composable functions should be side-effect-free

Example Side effects

  • Reading/Writing to/from a file
  • Accessing a database
  • Calling a REST API
  • …

Basic Elements

Column

Vertical stack of composables

Example

@Composable
fun Widget() {
    Column() {
        Text()
        Text()
        Button()
    }
}

Alignment/Arrangement

Column(
    horizontalAlignment = Alignment.CenterHorizontally,
    veritcalArrangement =  Arrangement.Center
) {
    ...
}

Arrangement - Where children should be positioned vertically

Alignment - Where children should be positioned horizontally

Row

Horizontal stack of composables

Alignment/Arrangment

Row(
    verticalAlignment = Alignment.CenterVertically,
    horizontalArrangement =  Arrangement.Center
) {
    ...
}

Arrangement - Where children should be positioned horizontally

Alignment - Where children should be positioned vertically

Type Safe

Arrangement and Alignment are typesafe. A compile-time error will occur if you attempt to use Vertical Arrangement on a Row and/or a Vertical Alignment on a Column.

Build complex layouts using a combination of Row and Column (and Box)

Column() {
    Text()
    Row() {
        Button()
        Text()
    }
    Button()
    ...
}

Box

Layer composables on top of each other

@Composable
fun BoxExample() {
    Box() {
        Text("Hello")
        Icon(...)
    }
}

Surface

Allows you to color/style the background on which other composables are drawn.

Spacer

Adds empty space between elements

@Composable
fun BoxExample() {
    Box() {
        Text("Hello")
    }
    Spacer(Modifier.width(20.dp))
    Box() {
        Text("Hello")
    }
}

Modifiers

Allows modification of a composable

  • Change the composable’s size, layout, behavior, and appearance

  • Make a composable clickable, draggable…

When creating your own composables, it is best-practice to include a modifier argument and provide a default

fun MyWidget(
    name: String,
    ...
    modifier = Modifer
)

Order matters

modifier
   .padding(16.dp)
   .fillMaxWidth()

is different than…

modifier
   .fillMaxWidth()
   .padding(16.dp)

Text Size

dp - Density independent pixels

sp - Scale independent pixels

Different devices have different screen densities which DP and SP account for. Otherwise, the same pixel size would look different on different devices.

Common modifiers

Modifier.padding(16.dp) // add padding *around* a composable
Modifier.fillMaxWidth()
Modifier.fillMaxHeight()
Modifier.weight(1f) // assign a relative weight to an element
