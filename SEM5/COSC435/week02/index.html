Basics

Syntax

  • No semi-colons (yay!)
  • Inline functions
  • Class-less functions
  • No new keyword
  • val, var
  • Otherwise, a lot like Java (classes, methods, packages)

Immutability

  • var - declare a variable as mutable
  • val - declare a variable as immutable (default)
  • Lots of collections are immutable by default listOf() vs mutableListOf()

Type inference

  • Local variable’s types do not need to be specified
  • Adding types is sometimes necessary, but usually not needed
  • Must declare types in function declarations
val hello = "Hello" // note no Type. inferred to be a String
val hello2: String = "Hello2" // this is also ok, but not needed

Nullable

  • By default, variables can’t be null
  • Use ? to denote a nullable variable
var something: String
var something = null  // error
var other: String? // nullable String (different than a String)!
var other = null // OK

Functions

  • Functions can be top-level (not inside a class)
  • Parameter types must be declared as well as return types
fun hello(str: String, i: Int): String {
    return "Hello" + i.toString()
}

Lambdas

  • Otherwise known as, inline functions
  • Much more concise than anonymous interface implementations
val someList = listOf(1, 2, 3, 4)
someList.filter { it > 2 } // returns a list of (3, 4)

Control flow

  • In addition to loops and if statements, Kotlin provides the very powerful: when
  • Like a switch statement on steroids
  • Can switch on values, types, …
val name: String? = "Joe"
when(name) {
  null -> println("Name is null")
  "Bob" -> println("Name is Bob")
  "Joe" -> println("Name is Joe")
  else -> println("Name is something else")
}

Object Orientation

Data Classes

  • Data classes make class declaration much more concise
  • Can’t be extended (class is final)
  • Auto-generates getters, setters, equals(), hashCode(), toString() and copy()
// Create a POJO with getters, setters, equals(), hashCode(),
// toString(), and copy() with a single line:
data class User(val name: String, val email: String)

Regular class

  • No generated toString(), …
  • Can be extended
// Create a POJO with getters, setters, equals(), hashCode(),
// toString(), and copy() with a single line:
class User(name: String, email: String) {
    val name: String
    val email: String
    constructor() {
       name = "Bob"
       email = "bob@gmail.com"
    }
}

Extending classes

  • Call parent constructors in the extending statement
  • Can only extend open classes (non-final)
